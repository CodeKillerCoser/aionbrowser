# Browser ACP v0.1.0

First public release of Browser ACP.

## Highlights

- Chrome side-panel extension for talking to local ACP-compatible agents.
- Native messaging host that starts and supervises the local daemon on demand.
- Agent configuration, discovery, session history, model loading, and model switching.
- Browser context handling through workspace-local temporary context files instead of large pasted prompt payloads.
- Login support for browser/OAuth style ACP methods and environment-variable API key methods.
- Page task templates for reusable selection actions.
- Debug drawer with panel, background, daemon, session, and raw ACP logs.

## Installation

Browser ACP has two pieces:

- `browser-acp-extension.zip`: the Chrome extension UI.
- A local Native Messaging host: the bridge that Chrome can launch to start the local daemon and ACP CLI agents.

Loading the extension zip alone is enough to open the side panel, but not enough to talk to local agents.

### 1. Load the extension

1. Download `browser-acp-extension.zip` from this release.
2. Unzip it.
3. Open `chrome://extensions/`.
4. Enable Developer mode.
5. Load the unzipped folder with Load unpacked.

### 2. Install the native host

For `v0.1.0`, the release does not yet include a standalone native host installer. Install it from this repository:

```bash
git clone https://github.com/CodeKillerCoser/aionbrowser.git
cd aionbrowser
pnpm install
pnpm build
pnpm install:native-host
```

If the native host installer cannot discover the extension ID, run:

```bash
pnpm --filter @browser-acp/native-host install:chrome-host -- --extension-id <extension-id>
```

## Notes

- This release is optimized for macOS and Chrome.
- The extension is distributed as an unpacked extension zip, not through the Chrome Web Store.
- The release zip contains the extension only; native host packaging will be improved in a future release.
- Agent model lists and login flows depend on each ACP CLI's current behavior.
